Goal

In a team of three, we developed a device aimed at monitoring a user's sleep patterns. Our approach involved monitoring the user's heart rate to determine if it fell below a specific threshold.

  • Data Collection: We set up a Raspberry Pi as a continuous data publisher, sending information to the broker without interruption.
  • Network Communication: We established a network using server and client configurations to create a system consisting of publishers, subscribers, and a broker. To transmit heart rate data to the broker, we utilized a Beaglebone Black. The broker not only collected this data but also awaited instructions from the subscribers.
  • Interface Design: Using Microsoft Visual Studio, we designed an interface that allowed subscribers to interact with the broker. The interface included the following functionalities:
    • Start API: This button connected the subscriber to the system's broker, enabling the real-time plotting of heart rate data on a chart.
    • Stop API: Clicking this button disconnected the subscriber from the broker and halted the plotting of heart rate data on the chart.
    • Stay Awake: Activating this function triggered the buzzer if the user's heart rate dropped below a specified threshold. The buzzer ceased when the heart rate returned above the threshold.
    • Wake Up: This function was employed to activate the alarm, irrespective of the user's heart rate.
    • Sleep: Sending this command turned off the Stay Awake function, preventing the buzzer from sounding, regardless of the user's heart rate.
  • Intergroup Communication: We configured the system to communicate with another group's broker, facilitating seamless interaction and data exchange between our systems.
